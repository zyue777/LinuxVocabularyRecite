## 一、五因子是什么？

### 🎯 通俗理解

想象你在炒股，想知道一只股票为什么涨或跌。Fama-French五因子模型告诉你，股票收益可以分解为5个因素：

| 因子代码 | 中文名称 | 通俗解释 | v2.0优化 |
|---------|---------|---------|---------|
| **MKT_RF** | 市场风险溢价 | 整个市场涨了多少？ | 无变化 |
| **SMB** | 规模因子 | 小公司比大公司多涨多少？ | 无变化 |
| **HML** | 价值因子 | 便宜股票比贵股票多涨多少？ | 无变化 |
| **RMW** | 盈利质量因子 | **现金流好的**公司比差的多涨多少？ | ⭐改用OCF |
| **CMA** | 投资因子 | 保守投资的公司比激进的多涨多少？ | ⭐改用NOA |

### 📊 数学表达

任何一只股票的超额收益可以用五因子模型解释：

```
股票超额收益 = α + β₁×MKT_RF + β₂×SMB + β₃×HML + β₄×RMW + β₅×CMA + ε

其中：
- α (Alpha): 超额收益，选股能力的体现
- β₁~β₅: 因子暴露度，衡量对各因子的敏感性
- ε: 残差，特异性风险
```

## 二、为什么需要v2.0优化？

### 🤔 v1.0的问题

#### 问题1: 营业利润(OP)容易被操纵

**v1.0计算**:
```python
RMW因子 = 营业利润 / 净资产
```

**问题**:
- ❌ 营业利润是账面数字，可能包含应收账款（还没收到钱）
- ❌ 中国企业财务粉饰常见，营业利润不可靠
- ❌ "纸面富贵"：利润很高但没有现金流

**实际案例**:
```
某公司2023年：
- 营业利润: 10亿元 ✨ （看起来很好）
- 经营现金流: -2亿元 ❌ （实际在烧钱）
→ 用OP计算会认为这是"盈利强"的公司，但实际现金流为负！
```

#### 问题2: 总资产增长(Inv)包含太多噪音

**v1.0计算**:
```python
CMA因子 = (今年总资产 - 去年总资产) / 去年总资产
```

**问题**:
- ❌ 总资产包含应收账款、金融资产等，不是真实投资
- ❌ 在建工程滞后计入，无法及时反映投资行为
- ❌ 金融企业的资产扩张与实业投资是两回事

**实际案例**:
```
某制造业公司：
- 去年总资产: 100亿（含应收账款20亿）
- 今年总资产: 120亿（应收账款增加到35亿）
- Inv = 20% ← 看起来激进投资
但实际上：
- 固定资产只增加了5亿（在建工程完工转入）
- 真实投资只有5% ← 应该是保守型
```

### ✅ v2.0的解决方案

#### 优化1: 使用经营现金流(OCF)

**v2.0计算**:
```python
RMW因子 = TTM经营现金流 / 净资产
```

**优势**:
- ✅ 现金流难以造假
- ✅ 反映真实的盈利质量
- ✅ 避免应收账款陷阱

#### 优化2: 使用经营性净资产(NOA)

**v2.0计算**:
```python
NOA = 有形资产（固定资产 + 在建工程 + 无形资产等）
CMA因子 = (今年NOA - 去年NOA) / 去年总资产
```

**优势**:
- ✅ 只看真实的资本开支
- ✅ 在建工程及时计入
- ✅ 排除金融资产干扰

## 四、核心概念：TTM（滚动12个月）

### 🤔 为什么需要TTM？

**问题**：公司业绩有季节性

```
某零售公司单季经营现金流：
Q1: 5亿    Q2: 8亿    Q3: 12亿   Q4: 20亿（春节效应）
         ↑ 直接用Q4数据会高估
```

**解决方案**：TTM = 过去4个季度的总和

```
TTM (Q4) = Q1 + Q2 + Q3 + Q4 = 5 + 8 + 12 + 20 = 45亿
→ 平滑了季节性波动
```

### 📈 TTM计算原理（重点！）

#### 问题：Tushare数据是累计值

```
Tushare返回的财务数据格式：
2024Q1: 10亿 ← 只有Q1
2024Q2: 25亿 ← Q1+Q2累计
2024Q3: 45亿 ← Q1+Q2+Q3累计
2024Q4: 63亿 ← Q1+Q2+Q3+Q4累计（年报）
```

#### 步骤1：转换为单季值

**代码位置**：`load_all_financials()` 函数，第91-94行

```python
# 对于Q2, Q3, Q4：单季值 = 累计值 - 上一季累计值
df['ocf_single_quarter'] = df.groupby(['ts_code', year])['n_cashflow_act'].diff()

# 对于Q1：单季值 = 累计值（就是Q1自己）
is_q1 = (df['end_date'].dt.month == 3)
df.loc[is_q1, 'ocf_single_quarter'] = df.loc[is_q1, 'n_cashflow_act']
```

**示例**:
```
原始数据:          单季值:
Q1: 10亿    →     10亿   (Q1累计值本身)
Q2: 25亿    →     15亿   (25 - 10)
Q3: 45亿    →     20亿   (45 - 25)
Q4: 63亿    →     18亿   (63 - 45)
```

#### 步骤2：计算TTM（滚动求和）

**代码位置**：第97行

```python
df['OCF_TTM'] = df.groupby('ts_code')['ocf_single_quarter'].rolling(
    window=4,      # 窗口大小=4个季度
    min_periods=4  # 至少需要4个季度才开始计算
).sum()
```

**示例**:
```
单季OCF:        TTM计算:
2023Q1: 8亿     ← 数据不足
2023Q2: 10亿    ← 数据不足
2023Q3: 12亿    ← 数据不足
2023Q4: 9亿     → TTM = 8+10+12+9 = 39亿
2024Q1: 10亿    → TTM = 10+12+9+10 = 41亿 (滚动更新)
2024Q2: 15亿    → TTM = 12+9+10+15 = 46亿
2024Q3: 20亿    → TTM = 9+10+15+20 = 54亿
2024Q4: 18亿    → TTM = 10+15+20+18 = 63亿
```

## 六、第二步：计算TTM财务指标

### 🧮 指标1：OCF_TTM（经营现金流TTM）

**代码位置**：第88-97行

#### 详细步骤

**Step 1: 排序和去重**
```python
df_cashflow = df_cashflow.sort_values(by=['ts_code', 'end_date']).drop_duplicates()
```
→ 按股票代码和日期排序，确保时间顺序正确

**Step 2: 计算单季现金流**
```python
# 同年内：当季累计值 - 上季累计值
df_cashflow['ocf_single_quarter'] = df_cashflow.groupby(
    ['ts_code', df_cashflow.end_date.dt.year]
)['n_cashflow_act'].diff()

# Q1特殊处理：Q1累计值就是单季值
is_q1 = (df_cashflow['end_date'].dt.month == 3)
df_cashflow.loc[is_q1, 'ocf_single_quarter'] = df_cashflow.loc[is_q1, 'n_cashflow_act']
```

**为什么要按年分组**？
```
错误示例（不按年分组）：
2023Q4累计: 630亿
2024Q1累计: 100亿
→ diff() = 100 - 630 = -530亿 ❌ 错误！

正确做法（按年分组）：
2023Q4: 单独计算
2024Q1: 新的一年，直接用累计值 = 100亿 ✅
```

**Step 3: 滚动求和计算TTM**
```python
df_cashflow['OCF_TTM'] = df_cashflow.groupby('ts_code')['ocf_single_quarter'].rolling(
    window=4,      # 4个季度
    min_periods=4  # 至少4个季度
).sum().reset_index(level=0, drop=True)
```

**可视化理解**:
```
时间轴:  Q1    Q2    Q3    Q4    Q1    Q2    Q3    Q4
单季:    10    15    20    18    10    15    20    18
         └─────────TTM1──────┘
               └─────────TTM2──────┘
                     └─────────TTM3──────┘
                           └─────────TTM4──────┘

TTM1 = 10+15+20+18 = 63
TTM2 = 15+20+18+10 = 63
TTM3 = 20+18+10+15 = 63
TTM4 = 18+10+15+20 = 63
```

### 🧮 指标2：NOA_Inv（经营性净资产投资因子）

**代码位置**：第102-121行

#### 详细步骤

**Step 1: 合并有形资产数据**
```python
df_balance = pd.merge(
    df_balance,
    df_fina[['ts_code', 'ann_date', 'end_date', 'tangible_asset']],
    on=['ts_code', 'ann_date', 'end_date'],
    how='left'  # 左连接，保留所有资产负债表记录
)
```

**Step 2: 定义经营性净资产(NOA)**
```python
df_balance['NOA'] = df_balance['tangible_asset'].fillna(0)
```

**NOA包含什么**？
```
有形资产 (tangible_asset) 包括：
├── 固定资产（厂房、设备）
├── 在建工程（建设中的项目）⭐关键！
├── 无形资产（土地使用权、专利）
└── 生产性生物资产（农业企业）

排除：
❌ 金融资产（股权投资、债券）
❌ 应收账款
❌ 存货
```

**为什么fillna(0)**？
- 银行股等金融企业没有tangible_asset → 填0
- 自动被CMA因子排除（这是合理的）

**Step 3: 获取一年前的NOA和总资产**
```python
df_balance['NOA_t_4'] = df_balance.groupby('ts_code')['NOA'].shift(4)
df_balance['assets_t_4'] = df_balance.groupby('ts_code')['total_assets'].shift(4)
```

**shift(4)的含义**:
```
当前数据:              shift(4)后:
2024Q1: NOA=100      2024Q1: NOA_t_4=80 (← 来自2023Q1)
2024Q2: NOA=105      2024Q2: NOA_t_4=85 (← 来自2023Q2)
2024Q3: NOA=110      2024Q3: NOA_t_4=90 (← 来自2023Q3)
2024Q4: NOA=115      2024Q4: NOA_t_4=95 (← 来自2023Q4)
```

**Step 4: 计算NOA投资因子**
```python
df_balance['NOA_Inv'] = (df_balance['NOA'] - df_balance['NOA_t_4']) / df_balance['assets_t_4']
```

**公式解释**:
```
NOA_Inv = (今年NOA - 去年NOA) / 去年总资产

分子：NOA的变化（真实资本开支）
分母：去年总资产（标准化，便于比较）

示例：
今年NOA: 120亿
去年NOA: 100亿  
去年总资产: 500亿
→ NOA_Inv = (120-100) / 500 = 4%

解读：公司用去年资产的4%进行了新投资
```

## 七、第三步：每月构建排序快照

### 📸 函数：`get_sorting_snapshot_monthly()`

**代码位置**：第203-252行

**目标**：每个月末，使用最新的已公告财务数据和当日市值，计算排序指标

### 时间线示例（关键！）

```
假设今天是 2024-01-31 (月末)

[时间轴]
2023Q3财报 → 2023-10-28公告
2023Q4财报 → 2024-01-28公告 ⬅ 刚公告3天
2024Q1财报 → 2024-04-28公告（未来，不能用）

[可用数据]
✅ 可用：截至2024-01-31已公告的最新财务数据 = 2023Q4
❌ 不可用：2024Q1数据还没公告（避免未来函数）

[调仓安排]
排序日：2024-01-31 用2023Q4数据计算因子
调仓期：2024-02-01 ~ 2024-02-29 持有这个组合
```

### 步骤详解

#### Step 1: 筛选已公告的最新财务数据

**代码**：第214-216行

```python
# 严格使用ann_date（公告日期）避免未来函数
df_fin_available = df_financials_ttm[df_financials_ttm['ann_date'] <= month_end]

# 每个股票取最新一份TTM报告
df_fin_snapshot = df_fin_available.sort_values('end_date').groupby('ts_code').last()
```

**示例**:
```
month_end = 2024-01-31

股票 000001.SZ 的财务数据：
end_date    ann_date    OCF_TTM    NOA_Inv
20230331    20230428    100亿      0.05
20230630    20230728    110亿      0.06
20230930    20231028    120亿      0.07
20231231    20240128    130亿      0.08  ← 选这个（最新的已公告）

股票 000002.SZ 的财务数据：
end_date    ann_date    OCF_TTM    NOA_Inv
20230331    20230425    80亿       0.03
20230630    20230725    85亿       0.04
20230930    20231025    90亿       0.04
20231231    20240203    95亿       0.05  ← 公告日在1月31日之后，不能用！
                                         ← 只能用20230930的数据
```

#### Step 2: 获取当月最后一天的市值

**代码**：第219-220行

```python
df_mv_snapshot = df_market_daily[df_market_daily['trade_date'] <= month_end]
```

- 用月末市值最能反映当前状态

## 九、第五步：计算每日因子收益

### 💰 函数：`calculate_daily_factors()`

**代码位置**：第321-432行

**目标**：用上个月末构建的组合，计算下个月每一天的因子收益

### 核心原理：市值加权收益率

**为什么用市值加权**？

```
❌ 错误：等权重
组合包含：茅台(市值2万亿) + 小股(市值20亿)
茅台涨1%，小股涨10%
等权收益 = (1% + 10%) / 2 = 5.5%
→ 不合理！小股权重太大

✅ 正确：市值加权
组合市值 = 2万亿 + 20亿 = 2.002万亿
茅台权重 = 2/2.002 = 99.9%
小股权重 = 0.02/2.002 = 0.1%
加权收益 = 99.9%×1% + 0.1%×10% = 1.009%
→ 合理！反映市场真实情况
```

### 市值加权函数

**代码位置**：第335-359行

```python
def get_vw_return(stocks_idx, date):
    """计算市值加权收益率
    
    参数:
        stocks_idx: 组合中的股票索引（如SH组合的所有股票）
        date: 计算哪一天的收益
        
    返回:
        该组合在该天的市值加权收益率
    """
    # 1. 获取当日收益率和昨日市值
    day_returns = df_returns.loc[date, stocks_idx].dropna()
    day_weights = df_prev_mv.loc[date, stocks_idx].dropna()
    
    # 2. 找出同时有收益率和市值的股票
    common_stocks = day_returns.index.intersection(day_weights.index)
    
    # 3. 市值加权
    total_mv = day_weights[common_stocks].sum()
    weighted_return = (day_returns[common_stocks] * day_weights[common_stocks]).sum() / total_mv
    
    return weighted_return
```

**为什么用昨日市值**？

```
今天买股票，用今天的价格（包含今天收益）计算权重是错的！

正确做法：
2024-02-01收盘: 用2024-01-31的市值决定权重
→ 权重是昨天确定的，今天按这个权重投资
```

**示例计算**:
```
SH组合（小市值+高B/M）在2024-02-01的收益:

股票      昨日市值   今日收益率   权重计算
000001.SZ  100亿     2%         100/(100+50+80) = 43.5%
000002.SZ   50亿     3%          50/(100+50+80) = 21.7%
000003.SZ   80亿     1%          80/(100+50+80) = 34.8%
总市值     230亿

加权收益 = 43.5%×2% + 21.7%×3% + 34.8%×1%
        = 0.87% + 0.65% + 0.35%
        = 1.87%
```

### 五因子计算公式

#### 因子1: HML（价值因子）

**代码**：第365-370行

**公式**:
```python
HML = (R_SH + R_BH) / 2 - (R_SL + R_BL) / 2
```

**逻辑**:
- 高B/M组合 = (小市值高B/M + 大市值高B/M) / 2
- 低B/M组合 = (小市值低B/M + 大市值低B/M) / 2
- HML = 高B/M - 低B/M

**经济含义**: 今天价值股比成长股多赚了多少？

**示例**:
```
2024-02-01:
R_SH (小市值高B/M) = 1.5%
R_BH (大市值高B/M) = 1.2%
R_SL (小市值低B/M) = 0.3%
R_BL (大市值低B/M) = 0.5%

HML = (1.5% + 1.2%) / 2 - (0.3% + 0.5%) / 2
    = 1.35% - 0.4%
    = 0.95%

解读：今天价值股比成长股多涨0.95%
```

#### 因子4: SMB（规模因子）

**代码**：第386-415行

**公式**（标准FF2015方法）:
```python
SMB = (SMB_BM + SMB_OP + SMB_Inv) / 3

其中:
SMB_BM = (R_SL + R_SM_BM + R_SH) / 3 - (R_BL + R_BM_BM + R_BH) / 3
SMB_OP = (R_SW + R_SM_OP + R_SR) / 3 - (R_BW + R_BM_OP + R_BR) / 3
SMB_Inv = (R_SA + R_SM_Inv + R_SC) / 3 - (R_BA + R_BM_Inv + R_BC) / 3
```

**逻辑**:
- 在3个维度（B/M, OCF_B, NOA_Inv）分别计算小市值-大市值
- 取3个维度的平均

**为什么这样算**？
- 确保SMB不受其他因子影响
- 例如：小市值股票可能同时是高B/M，这样算可以分离出纯粹的规模效应

**示例**:
```
B/M维度:
Small平均 = (R_SL + R_SM_BM + R_SH) / 3 = (1.5% + 1.8% + 2.0%) / 3 = 1.77%
Big平均 = (R_BL + R_BM_BM + R_BH) / 3 = (0.5% + 0.8% + 1.2%) / 3 = 0.83%
SMB_BM = 1.77% - 0.83% = 0.94%

OCF_B维度:
Small平均 = (R_SW + R_SM_OP + R_SR) / 3 = 1.6%
Big平均 = (R_BW + R_BM_OP + R_BR) / 3 = 0.9%
SMB_OP = 1.6% - 0.9% = 0.7%

NOA_Inv维度:
Small平均 = (R_SA + R_SM_Inv + R_SC) / 3 = 1.7%
Big平均 = (R_BA + R_BM_Inv + R_BC) / 3 = 1.0%
SMB_Inv = 1.7% - 1.0% = 0.7%

SMB = (0.94% + 0.7% + 0.7%) / 3 = 0.78%

解读：今天小市值股票比大市值多涨0.78%
```

## 十一、常见问题FAQ

### Q1: 为什么v2.0更好？

**A**: 针对A股特点优化

| 维度 | v1.0问题 | v2.0解决方案 |
|------|---------|-------------|
| 盈利质量 | 营业利润易造假 | 现金流难造假 ✅ |
| 投资衡量 | 总资产噪音大 | 只看真实资本开支 ✅ |
| A股适配 | 标准方法 | 实战优化 ✅ |

---

### Q2: 银行股为什么被CMA因子排除？

**A**: 这是合理的！

```
银行业务特点:
├── 主要资产: 贷款、债券（金融资产）
├── 有形资产: 办公楼、IT设备（占比<10%）
└── 投资方式: 发放贷款，而非购买固定资产

→ tangible_asset对银行无意义
→ CMA因子排除银行股是正确的
→ 银行股仍参与其他4个因子
```

详见: [银行股与FF5因子影响分析.md](银行股与FF5因子影响分析.md)

---

### Q3: 为什么要用TTM而不是单季度？

**A**: 平滑季节性

```
零售业现金流:
Q1: 10亿  Q2: 15亿  Q3: 20亿  Q4: 50亿（春节）

如果用单季:
Q4公司被标记为"强现金流" ← 不准确（季节性）

如果用TTM:
TTM = 10+15+20+50 = 95亿 ← 准确（全年视角）
```

---

### Q4: 为什么月度调仓而不是季度或年度？

**A**: 平衡及时性和稳定性

| 频率 | 优点 | 缺点 |
|------|------|------|
| 日度 | 最及时 | ❌交易成本高、噪音大 |
| 月度 | ✅及时+稳定平衡 | 略微滞后 |
| 季度 | 稳定 | ❌反应太慢 |

---

### Q5: 为什么用2×3排序而不是3×3或5×5？

**A**: Fama-French标准方法

```
2×3的优势:
✅ 每个组合股票数量充足（避免小样本）
✅ 因子区分度足够（30%极端组）
✅ 学术界标准，便于对比

3×3或5×5:
❌ 组合太多，每个组合股票太少
❌ 过度拟合风险
```

---

### Q6: 如何使用五因子做回归分析？

**A**: 示例代码

```python
import pandas as pd
from sklearn.linear_model import LinearRegression

# 1. 读取FF5因子
df_ff5 = pd.read_parquet('quant_data_center/factors/fama_french_5/ff_5_factors_daily.parquet')

# 2. 读取股票收益率
df_stock = pd.read_parquet('quant_data_center/stock/daily_hfq/000001.SZ.parquet')
df_stock['return'] = df_stock['close'].pct_change() * 100

# 3. 合并
df = df_stock.merge(df_ff5, on='trade_date', how='inner')

# 4. 回归
X = df[['MKT_RF', 'SMB', 'HML', 'RMW', 'CMA']].values
y = df['return'].values

model = LinearRegression()
model.fit(X, y)

print(f"Alpha: {model.intercept_:.4f}%")
print(f"Beta MKT: {model.coef_[0]:.3f}")
print(f"Beta SMB: {model.coef_[1]:.3f}")
print(f"Beta HML: {model.coef_[2]:.3f}")
print(f"Beta RMW: {model.coef_[3]:.3f}")  # v2.0: 现金流质量暴露
print(f"Beta CMA: {model.coef_[4]:.3f}")  # v2.0: 投资风格暴露
```

### Q7: 因子数据从什么时候开始？

**A**: 2012-11-01起

```
原因:
├── TTM需要4个季度数据
├── NOA_Inv需要再往前4个季度（计算同比）
├── 共需8个季度 = 2年历史数据
└── 从2012年10月开始计算，11月开始有完整因子

数据积累:
2010-2011: 积累8个季度财务数据
2012-01至2012-10: 积累初期TTM数据
2012-11起: 正式输出因子
```

### Q8: 为什么要用公告日期(ann_date)而不是报告期(end_date)？

**A**: 避免未来函数！

```
错误示例(用end_date):
2024-01-31排序
→ 用2023Q4数据（end_date=20231231）
→ 但Q4财报在1月28日才公告
→ 1月1日至27日用了未来信息 ❌

正确做法(用ann_date):
2024-01-31排序
→ 只用ann_date≤20240131的数据
→ 1月28日前用Q3数据，28日后用Q4数据
→ 严格遵守时间顺序 ✅
```

### Q10: v2.0因子与v1.0相关性如何？

**A**: 中等相关，但有显著差异

```
预期相关性:
MKT_RF: 1.00 (完全相同)
SMB: 0.95+ (基本相同，只是RMW和CMA的SMB组件略有不同)
HML: 0.95+ (完全相同)
RMW: 0.6-0.8 (OP vs OCF有差异)
CMA: 0.5-0.7 (Inv vs NOA_Inv差异较大)

差异来源:
1. 现金流vs营业利润：账面差异
2. 有形资产vs总资产：统计口径不同
3. 股票池略有差异（部分股票NOA缺失）
```
